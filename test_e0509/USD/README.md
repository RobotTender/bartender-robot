# test_e0509 USD 메모

이 문서는 `test_e0509/USD` 내부 에셋의 "실행/배치에 필요한 핵심 정보"만 정리합니다.
전개도/상세 모델링 이력은 포함하지 않습니다.
아래의 "사용 task/학습파일" 구분은 현재 저장소 코드 기준으로 정리합니다.

## 1. table_hole.usd

- 파일: `USD/table_hole.usd`
- 역할: 학습 환경에서 E0509 로봇이 올라가는 워크스페이스(작업대) 기본 구조
- 기준 단위: mm

사용 task/학습파일:

- `Grip Bottle`: `grip_bottle_env.py`
- `Move Bottle`: `move_bottle_env.py`
- 용도: 로봇이 올라가는 워크스페이스 본체

핵심 치수/배치:

- 테이블 상판 높이: `700 mm`
- 로봇 마운트 블럭 높이: `30 mm` (상판 위)
- 로봇 베이스 높이: `730 mm` (월드 ground 기준)
- 테이블 상판 크기: `1200 x 568 mm`
- 베이스 기준 상판 한 끝점 좌표: `(-343, -290)` (mm)

상판 꼭지점 좌표(mm):

- `P1 = (-343, -290)`
- `P2 = (857, -290)`
- `P3 = (857, 278)`
- `P4 = (-343, 278)`

상판 XY 배치(월드 기준, 회전 없음):

```text
y+
^
|   P4(-343,278) ---------------- P3(857,278)
|   |                                   |
|   |                                   |
|   P1(-343,-290) --------------- P2(857,-290)  -> x+
```

좌표계/정렬:

- 월드 z축과 로봇 베이스 z축이 일치하도록 오프셋 정렬
- 월드 ground 기준 z 높이(베이스 730 mm)로 배치

비고:

- 현재 학습 환경은 `table_hole`을 워크스페이스 기준 에셋으로 사용
- 코드 변수명에서 `table_*`가 보여도 프로젝트 의미상 "선반/작업대"로 해석 가능

## 2. tables_3.usd

- 파일: `USD/tables_3.usd`
- 역할: 실제 환경의 병 배치 선반을 단순화해 학습용으로 옮긴 에셋
- 기준 단위: mm

사용 task/학습파일:

- `Grip Bottle`: `grip_bottle_env.py`
- `Move Bottle`: `move_bottle_env.py`
- 용도: 병 배치 맥락, 선반 위치, 충돌 공간을 함께 반영하는 선반/배경 에셋

배경:

- 실제 환경은 테이블 5개를 쌓아 만든 선반 구조였음
- 학습 환경에서는 계산 비용과 모델 복잡도를 줄이기 위해 3개만 구현함
- 하단 지지 구조도 실제보다 단순화된 형태로 반영되어 있음

핵심 치수/배치:

- 물건이 실제로 놓이는 선반 면 높이: `1300 mm`
- 물건이 놓이는 선반 면 크기: `724 x 300 mm`
- 모델 내부 배치 오프셋: `z축 기준 180도 회전`, `(-55, 1120)` 이동
- `y = 1120 mm`는 로봇 베이스와 선반 시작점 거리 `520 mm`에 테이블 길이 `600 mm`를 더해 맞춘 값임

구조/사용 메모:

- 이 에셋은 실제 바텐더 환경의 선반 위치와 병 배치 영역을 학습 환경에서 맞추기 위한 기준 구조물임
- 로봇 쪽 선반 가장자리에 `5 mm` 턱이 있어, 선반 위 물체를 로봇 방향으로 끌어오거나 이동시키려면 약간 들어 올리는 동작이 필요함
- 따라서 파지 후 이동 정책에서는 단순 XY 이동만이 아니라, 턱을 넘기기 위한 미세 리프트나 충돌 회피를 함께 고려해야 함

비고:

- `tables_3`는 실제 환경을 그대로 복제한 모델이라기보다, 학습 안정성과 계산 비용을 고려해 단순화한 버전임
- 배치 오프셋이 에셋 내부에 이미 반영되어 있으므로, task에서는 이 값을 기준으로 위치를 해석하면 됨

## 3. e0509/ 로봇 에셋 묶음

- 경로: `USD/e0509/`
- 역할: Doosan E0509 암과 RH-P12-RN(A) 그리퍼 관련 로봇 USD 묶음

실행 기준:

- `e0509_model.usd`
  - 현재 task 코드가 직접 로드하는 로봇 모델
  - `Grip Bottle`, `Move Bottle` 모두 이 파일을 기준으로 로봇을 생성함
  - 현재 프로젝트에서 "실제로 쓰는 로봇 USD"는 이 파일이라고 보면 됨
  - 파일 크기(`~5.8 MB`)와 구성상, 암과 그리퍼가 결합된 실행용 모델이다
  - 내부 구성상 같은 폴더의 `gripper.usd`를 함께 참조하는 결합형 자산으로 보는 편이 맞다
  - 따라서 현재 실행 기준에서는 `e0509_model.usd`만 따로 보는 것이 아니라, 같은 경로의 `gripper.usd`와 함께 유지해야 한다

구성 자산 기준:

- `e0509.usd`
  - `e0509_model.usd`와 별도로 남아 있는 조립형 루트 자산
  - 현재 task 코드의 직접 로드 대상은 아님
  - 사용자가 확인한 것처럼 `e0509_model.usd`는 이 파일이 없어도 동작할 수 있으므로, 현재 실행 필수 파일로 보지 않음
  - 대신 로봇 자산 구조를 분리해서 이해할 때 참고용으로 남겨두는 파일이다

- `configuration/`
  - `e0509.usd`와 함께 묶여 있는 분리형 구성 자산 폴더
  - 현재 task 실행에서 직접 로드되는 주력 경로는 아니지만, 로봇 자산을 base/robot/physics/sensor 단위로 나눠 둔 자료다

`configuration/` 세부 역할:

- `configuration/e0509_base.usd`
  - 가장 큰 파일(`~5.9 MB`)로, 로봇 기본 형상과 기반 구조를 담는 핵심 자산
- `configuration/e0509_physics.usd`
  - 물리 설정 레이어
- `configuration/e0509_robot.usd`
  - 로봇 조립 레이어
- `configuration/e0509_sensor.usd`
  - 센서 관련 레이어

정리 기준:

- 현재 프로젝트 실행만 기준으로 보면 핵심 파일은 `e0509_model.usd`다
- `e0509.usd + configuration/`는 "현재 실행 필수"라기보다 "분리형 로봇 자산 구조를 설명하는 참고 묶음"으로 보는 편이 정확하다
- 따라서 이 문서에서는 `e0509_model.usd`를 실행용 모델로, `e0509.usd + configuration/`를 참고용 구성 자산으로 구분해서 설명한다

- 기준 단위: mm
